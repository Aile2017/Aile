# B2E スクリプト仕様の改善案

zpaq.b2e の実装を通じて、B2E スクリプト言語の改善が必要な点が明らかになりました。

## 1. セクション存在時の未実装ハンドリング

**現状の問題：**
- セクションが存在するだけで対応メニューが有効になる
- セクション内容が空でも有効と判定される
- zpaq のように削除機能のない場合、削除メニューをグレイアウトできない

**改善案：**
- セクション存在 ≠ メニュー有効と分離する
- セクション内容が空または実装不完全な場合、そのメニューをグレイアウト
- または明示的な「未実装マーク」を定義（例：`(unimplemented)` 等）

**実装例：**
```lisp
; セクションは存在するがメニューは無効にしたい場合
delete:
 (unimplemented)
```

## 2. B2E スクリプトへのコメント機能追加

**現状の問題：**
- スクリプトにコメントが書けない
- 複雑なロジックの意図がわからない
- zpaq.b2e でも出力フォーマットの詳細を注記したい

**仕様：**
- `;` 以降、行末までをコメントとみなす（Lisp 標準と同じ）
- 行のどの位置にあっても有効（行頭・行末・式の後ろ）
- 文字列リテラル `"..."` 内の `;` はリテラルの一部として扱われる

**文法例：**

```lisp
load:
 ; zpaq64.exe の圧縮レベルは -m0 (fastest) から -m5 (best)
 (name zpaq64.exe)
 (type zpaq Store *Method1 Method2 Method3 Method4 Method5)

encode:
 (if (method 2) (cmd a (arc.zpaq) (listr) -m1)) ; デフォルトメソッド
```

注: zpaq はレスポンスファイル（`@file`）を圧縮時にサポートしない。`(listr)` でファイル名を直接展開する。
また zpaq の位置引数規則により、アーカイブ名は必ず最初（`a` の直後）に置く必要がある。

**実装：** `kl_rythp.cpp` の `split_tonext()` で処理。トークン間に `;` を検出したら行末までスキップする。

## 実装の優先度

1. **低** — コメント機能（利便性向上、既存機能への影響なし）
2. **中** — セクション未実装ハンドリング（UX改善）
