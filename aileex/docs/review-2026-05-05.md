# Code Review Findings (2026-05-05)

Comprehensive source and documentation review findings of inconsistencies and implementation issues.
Follow-up notes for later discussion. Priority is subjective based on implementation impact.

## 1. Documentation / Specification vs Implementation Inconsistencies

### 1-1. Extensions accepted but cannot be opened (Medium)

- `src/main.cpp:8-10` `kArchiveExts[]` lists `wim`, `lzma`, `lzh`, `arj` as archive detection targets.
- Also listed in supported format in `docs/specification.md` and `README.md`.
- Meanwhile, `SevenZip::FormatToInGuid()` (`src/SevenZip.cpp:224`) doesn't handle these extensions, falling back to `return CLSID_Format_7z`.
- Result: 7z handler returns `S_FALSE` on magic mismatch, UI shows "Failed to open archive".

**Resolution options:**
- Add `CLSID_Format_Wim (0xE6)`, `CLSID_Format_Lzh (0x06)`, `CLSID_Format_Arj (0x04)`, `CLSID_Format_Lzma (0x0A)` to `sdk/7zip/Archive/IArchive.h`, add branches in `FormatToInGuid`.
- Or remove from `kArchiveExts[]` and spec.md/README if not implementing.

### 1-2. 7z header encryption enabled in UI but ineffective (High)

- `res/AileEx.rc:37` `IDC_ENCRYPT_HDR` checkbox enabled for 7z in `CompressDlg.cpp:228` via `EnableWindow(..., is7z)`, user can check.
- `OnOK` reads value into `m_params.encryptHeaders`.
- But `SevenZip::Compress()` (`src/SevenZip.h:41`) signature has **no** `encryptHeaders` argument. No `he=on` equivalent in `SevenZip.cpp:1095-1133` `setProps`.
- Result: Checking "Header encryption" in 7z has no effect. Contradicts spec.md "7z has header encryption option".

**Resolution:**
- Add `bool encryptHeaders` to `SevenZip::Compress`, push `he` / `VT_BSTR("on")` to setProps names/vals.
- Update callers (`App.cpp:182`, `MainWindow.cpp:854`).

### 1-3. RAR compression via drag-drop doesn't reflect advanced parameters (Medium)

- `MainWindow::OnDropFiles` (`MainWindow.cpp:402-411`) fills `params.dictSize` etc., but **not** `rarDictSize` / `rarSolid` / `rarThreads` / `rarRecoveryPct` / `rarSplitVolume` / `rarExtra`.
- `OnAddFiles` (`MainWindow.cpp:716-731`) fills them correctly.
- Result: Drag-drop → RAR selection passes empty RAR advanced values to CompressDlg. Post-OK save (`MainWindow.cpp:417-426`) also doesn't save RAR fields, risk overwrite.

**Resolution:**
- Match initialization block to `OnAddFiles` (preferably create common helper).

### 1-4. No cancel path for RAR compression via command line (Medium)

- RAR path in `App::RunCompressMode` (`App.cpp:152-167`) message loop handles `WM_APP_DONE` and `WM_APP_PROGRESS`, but **neither checks `sink->IsCancelled()` nor calls `rarProc.Cancel()`**.
- `MainWindow::OnCompress` `runMsgLoop` (`MainWindow.cpp:793-811`) has cancel mechanism via `cancelFn` calling `rarProc.Cancel()`, asymmetric.
- Result: Pressing cancel on progress dialog of CLI-launched compression doesn't stop rar.exe.

**Resolution:**
- Unify RAR path in `App::RunCompressMode` to same structure as `MainWindow::OnCompress`, or create shared function.
- Since `progDlg.SetSink(sink)` is called, loop should check `sink->IsCancelled()` → `rar.Cancel()`.

### 1-5. rar.exe exit code not reflected in UI (High)

- `RarProcess::StdoutReaderThread` (`RarProcess.cpp:226-266`) always Posts **wParam=0 (= S_OK) fixed** at pipe EOF via `PostMessageW(ctx->hwnd, ctx->doneMsg, 0, 0)`.
- Never calls `GetExitCodeProcess`, so rar.exe failure still shows "Complete" in UI.
- `WinrarWaiterThread` (`RarProcess.cpp:79-94`) gets ExitCode and converts to `HRESULT_FROM_WIN32(ERROR_CANCELLED)`, asymmetric.

**Resolution:**
- At end of `StdoutReaderThread`: `WaitForSingleObject(hProcess, INFINITE)` → `GetExitCodeProcess` → Post `(ec == 0) ? S_OK : E_FAIL`.
- Requires `HANDLE hProcess` in `ReaderCtx` (DuplicateHandle in `Compress`).

### 1-6. unrar.dll modification time off by timezone (Low-Medium)

- `UnrarDll::ListArchive` (`UnrarDll.cpp:96`) stores `DosDateTimeToFileTime(..., &it.mtime)` directly.
- `DosDateTimeToFileTime` output is **local FILETIME** (DOS time itself is local).
- Meanwhile `MainWindow::PopulateList` (`MainWindow.cpp:1057`) and `InfoDlg::FormatFileTime` (`InfoDlg.cpp:11-12`) assume **UTC FILETIME**, applying `FileTimeToLocalFileTime`.
- Result: In non-UTC locales, RAR file times are off by timezone offset (JST: +9 hours).

**Resolution:**
- In `UnrarDll.cpp:96`: take `DosDateTimeToFileTime` output as temp FILETIME, convert with `LocalFileTimeToFileTime(&local, &it.mtime)` to UTC before storing.

### 1-7. Settings dialog "burns in" auto-detected path (Medium)

- `SettingsDlg::OnInit` (`SettingsDlg.cpp:113-118`) shows auto-detected value if empty.
- `SettingsDlg::OnOK` (`SettingsDlg.cpp:166-170`) saves displayed string directly via `Set*Path`.
- Result: Even if user touches nothing and just clicks OK, auto-detected absolute path is written as fixed value to INI. Later 7-Zip upgrades changing paths still use old path.

**Resolution:**
- In `OnInit`, remember if value is auto-detected; if unedited, save empty string on OK.
- Or separate display-only label + edit control (UX change).

### 1-8. Documentation side expressions
- `docs/specification.md:60-63` "unrar.dll always full extract" not reflected in `README.md:67` (README generalizes "selective extract possible").
- `docs/build.md:50` table would benefit from MSVC 18/2026 path (CLAUDE.md value) note.
- Mismatch between "supported formats" in CLAUDE.md and `FormatToInGuid` implementation (same as 1-1).

## 2. Potential Implementation Bugs

### 2-1. Existing file corrupted on compression failure (Medium)

- `SevenZip.cpp:1146-1152` `COutFileStream::Create(outPath)` uses `CREATE_ALWAYS`.
- If later `archive->UpdateItems` fails, existing same-named file already truncated to 0 bytes.

**Resolution:**
- Write to temp file (e.g., `outPath + L".part"`), only `MoveFileEx(..., MOVEFILE_REPLACE_EXISTING)` on success.

### 2-2. CExtractCallback `m_currentOut` reset leak (Low)

- Around `SevenZip.cpp:709-735` `GetStream`/`SetOperationResult`.
- `m_currentOut` not reset on skip cases (`askExtractMode != kExtract`, directories).
- On consecutive calls, may re-Release previous `m_currentOut`.
- Not reproducible with current 7z.dll behavior but fragile.

**Resolution:**
- `GetStream` start: reset `m_currentOut = nullptr; m_currentIsDir = false; m_currentItemIndex = -1;`.

### 2-3. No command line escaping for passwords (Low)

- `RarProcess.cpp:128-132`: `cmd += L" -hp" + pw` / `L" -p" + pw`.
- Passwords with spaces, `"`, `^` cause CreateProcessW parsing failure.
- Safer via password file `-p@filename` method.

**Resolution:**
- Add command line escaping (quote wrapping) for passwords.

### 2-4. CreateProcessW bInheritHandles=TRUE (Low)

- `RarProcess.cpp:192` (Console mode) sets `bInheritHandles=TRUE`.
- All parent process inheritable handles inherited.
- Better to use `STARTUPINFOEXW` + `PROC_THREAD_ATTRIBUTE_HANDLE_LIST` to limit to stdout handle only.

**Resolution:**
- Use `STARTUPINFOEXW` with handle list.

### 2-5. MainWindow::OnProgress / OnDone are effectively dead code (Low, cleanup)

- Nested MSG loop in `MainWindow::OnExtract` (`MainWindow.cpp:646-663`) and `OnCompress::runMsgLoop` (`MainWindow.cpp:793-811`) absorb `WM_APP_PROGRESS` with `continue`, so `WM_APP_PROGRESS` → `OnProgress` path in `HandleMsg` (`MainWindow.cpp:231-233`) never reached.
- Keeping as defensive code OK; if kept, add `// fallback` comment to clarify intent.

**Resolution:**
- Keep but add clarifying comment.

### 2-6. No password support on archive Open (Medium)

- Opening 7z with encrypted header causes `COpenCallback::CryptoGetTextPassword` (`SevenZip.cpp:98`) to return empty BSTR, Open fails.
- No password input UI exists.

**Resolution:**
- If `OpenArchive` returns failure HRESULT (like `E_ACCESSDENIED`) or specific pattern, show password dialog and retry.
- API already has `password` argument in `SevenZip.h:30`, so add UI → pass path.

**Status:** ✅ Implemented in `6652566`: Added `IDD_PASSWORD` dialog, password retry on Open failure.

### 2-7. Settings::Save precondition (Low)

- `Settings.cpp:54` `Save` depends on `m_iniPath` being set via `Load` (doesn't call `BuildIniPath`).
- Currently always called as `Load` → `Save` in `App::Init`, but relies on call order.
- Could add defensive `if (!m_iniPath[0]) BuildIniPath();` at `Save()` start (maintain `const`).

**Resolution:**
- Add defensive path building in Save.

### 2-8. CompressDlg display flicker (Minimal)

- `src/CompressDlg.cpp:55` `kMethodsRar` "Normal" row label has double space in one line. Cosmetic.

**Resolution:**
- Fix spacing.
