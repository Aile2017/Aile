# Rythp 言語仕様

Rythp は B2E スクリプトを記述するための最小限の Lisp 方言です。`kilib` の `kiRythpVM` として実装されており、B2E 向けの拡張関数群は `CB2eCore::exec_function`（`ArcB2e.cpp`）で提供されています。

---

## 1. 字句構造

### 1.1 文字コード

スクリプトファイルは UTF-8 または Shift-JIS で記述できます。VM 内部は UTF-16（`wchar_t`）で動作します。

### 1.2 ホワイトスペース

スペース、タブ、CR、LF は区切り文字として機能します。何個あっても一つの区切りとみなします。

### 1.3 コメント

`;` から行末までがコメントです。

```
; これはコメント
(cmd a (arc.rar))  ; 行末コメントも可
```

コメントはトークン境界にのみ記述できます。引用符（`"`）で囲まれた文字列内の `;` はコメントになりません。

### 1.4 トークンの種類

Rythp のトークンは次の 2 種類だけです。

| 種類 | 形式 | 例 |
|---|---|---|
| **リテラル** | `(` 以外で始まる空白なし文字列、または `"..."` | `hello`, `-m5`, `0`, `"hello world"` |
| **式** | `(` で始まる対応する `)` までの入れ子 | `(arc.7z)`, `(if (method 1) ...)` |

`"..."` で囲んだ場合、内部の空白もリテラルの一部になります。文字列リテラルは **複数行にまたがることができません**。

## 2. 変数

### 2.1 変数名

変数名は **1 文字の ASCII コード** です。使用可能な文字は `a`–`z` と `A`–`Z` の計 52 文字です。

内部的には `ele[256]` という配列のインデックスとして使われます。変数名に複数文字を書いた場合、先頭の 1 文字だけが有効です。

```
(let abc 10)   ; 変数 'a' に 10 を代入 ('b', 'c' は無視される)
(let a 10)     ; 同じ意味
%abc           ; 変数 'a' の値を展開 → "10"
```

### 2.2 変数の展開

リテラルトークン内で `%x`（`x` は変数名）と書くと、その変数の値に置き換えられます。

```
(let o "-mx=9")
(cmd a -t7z %o (arc.7z))    ; → 7zz.exe a -t7z -mx=9 <archive>
```

`%` 展開は連結として機能します。

```
(let p "mypass")
(cmd a -p%p (arc))          ; → 7zz.exe a -pmypass <archive>
```

### 2.3 特殊文字エスケープ

以下の文字は起動時に特定の変数として初期化されています。

| 書き方 | 展開後 | 意味 |
|---|---|---|
| `%%` | `%` | パーセント記号そのもの |
| `%"` | `"` | ダブルクォート |
| `%(` | `(` | 開き括弧 |
| `%)` | `)` | 閉じ括弧 |
| `%/` | 改行（`\n`） | 改行文字 |

これらは変数として実装されているため、`(let / "\n")` で上書きすることはできません（同名変数への再代入が可能なため上書きされます）。なお `%/` は改行をコマンド引数に埋め込みたい場合に使います。

## 3. 式と評価

### 3.1 式の構文

```
(name arg1 arg2 arg3 ...)
```

`name` は関数名（それ自体もリテラルまたは入れ子式で評価されます）、`arg1`〜`argN` は引数です。

引数は **遅延評価** です。関数の内部で明示的に評価（`getarg` を呼ぶ）したときに初めて評価されます。これは `if` や `while` の条件/ボディが実行回数に応じて評価される理由です。

### 3.2 入れ子

引数として別の式を書けます。

```
(cmd a -t7z %o -r0 (arc.7z) (resp@ (listr)) -scsUTF-8)
```

`(arc.7z)` と `(resp@ (listr))` は、`cmd` が引数を評価する時点でそれぞれ評価されます。

### 3.3 返り値

すべての式は文字列を返します。何も返さない関数（`exec_function` が値をセットしない場合）は空文字列 `""` を返します。

### 3.4 真偽値

Rythp にブーリアン型はありません。整数の `0`（ゼロ）が**偽**、それ以外の整数が**真**として扱われます。

```
(if (method 2) (let o "-mx=9"))    ; (method 2) が "1" または "0" を返す
```

`if` や `while` は返り値を `getInt()` で整数に変換して判定します。

## 4. 型システム

Rythp のすべての値は **文字列** です。ただし算術演算や比較では整数として解釈されます。

### 4.1 整数変換（`getInt()`）

文字列を整数として読み取ります。

- 先頭の `-` はマイナス符号として扱います。
- 続く文字が `0`–`9` の間は 10 進数として読み取ります。
- **最初の非数字文字で読み取りを停止し、それ以前の部分を返します**。
- 空文字列、または先頭が非数字（`-` でもない）の場合は `0` を返します。

```
"42"      → 42
"-7"      → -7
""        → 0
"3.14"    → 3      (小数点で停止)
"1,234"   → 1      (カンマで停止)
"hello"   → 0
```

### 4.2 整数文字列の判定（`isIntStr()`）

`=` と `!` の比較演算が「整数として比較するか、文字列として比較するか」を決定するために使う内部関数です。

文字列を構成する文字が**すべて** `0`–`9`、`,`、`-` のいずれかであるとき `true` を返します。

```
"42"       → true  （整数比較）
"1,234"    → true  （整数比較、カンマを含む）
"-5"       → true  （整数比較）
"3.14"     → false （小数点を含む → 文字列比較）
"hello"    → false （文字列比較）
""         → true  （全文字が条件を満たす → 整数比較、getInt()=0）
```

> **注意**: `"1,234"` は `isIntStr()` が `true` を返しますが、`getInt()` はカンマで止まるため `1` を返します。例えば `(= "1,234" "5,678")` は整数比較になり、双方とも `getInt()` = 1 ≠ 5 なので `"0"` を返します。カンマ付き数値の比較は実用上は `0` との比較（ゼロか否か）にのみ使用してください。

## 5. 組み込み関数リファレンス

### 5.1 制御フロー

#### `(exec stmt1 stmt2 ...)`

引数を左から順に評価します。最後の引数の値を返します。

セクションの本体全体は `(exec ...)` で自動的にラップされるため、複数のトップレベル式を並べて記述できます。

```
encode:
 (let o "-mx=9")           ; これらは
 (cmd a (arc.7z) %o)       ; 暗黙的に (exec ...) でラップされる
```

#### `(if cond then [else])`

`cond` を評価し、`getInt()` が 0 でなければ `then` を評価して返します。0 であれば `else` を評価して返します（`else` がなければ返り値は `""` のまま）。

```
(if (method 1) (let o "-m0=Copy") (let o "-mx=9"))
```

#### `(while cond body)`

`cond` を評価し、`getInt()` が 0 でない限り `body` を繰り返し実行します。

> **実装上の注意**: `cond` と `body` の引数文字列は評価前にコピーされます。これは `eval()` が引数バッファを in-place で書き換えるためです。スクリプト作者には透過的ですが、ループ内での書き換えは考慮不要です。

```
(let i 1)
(while (< %i 5)
  (exec
    (cmd a -v%iM (arc.7z) (listr))
    (let i (+ %i 1))))
```

---

### 5.2 変数操作

#### `(let var val1 [val2 ...])`

引数 `val1`, `val2`, ... をすべて評価し、結果を連結したものを変数 `var` に代入します。代入した値を返します。変数名の先頭 1 文字だけが有効です。

```
(let o "-mx=9")             ; o = "-mx=9"
(let p "pass" "word")       ; p = "password"
(let f (arc.tar))           ; f = (arc.tar) の評価結果
```

値引数が 0 個の場合は空文字列を代入します。

---

### 5.3 算術演算

すべての算術演算はオペランドを `getInt()` で整数に変換してから計算し、結果を 10 進数文字列で返します。

#### `(+ A [B ...])`

全引数の合計を返します。引数なしは `"0"` を返します。

```
(+ 3 4)           → "7"
(+ %i 1)          → i+1 の文字列
(+ 1 2 3)         → "6"
```

#### `(- A [B ...])`

`(- A)` は `-A`（符号反転）を返します。  
`(- A B C ...)` は `A - B - C - ...` を返します。  
引数なしは `"0"` を返します。

```
(- 10 3)          → "7"
(- %x)            → x の符号反転
(- 10 3 2)        → "5"
```

#### `(* A [B ...])`

全引数の積を返します。引数なしは `"1"` を返します。

```
(* 3 4)           → "12"
(* (= %x 1) 5)    ; x==1 のとき 5 (= 1*5)、そうでなければ 0 (= 0*5)
```

`+` が論理 OR のように、`*` が論理 AND のように使えます（ただし真の値が 1 でない場合は結果が変わります）。

#### `(/ A B)`

`A ÷ B`（切り捨て整数除算）を返します。`B == 0` のとき `A` を返します（ゼロ除算はエラーにならない）。

#### `(mod A B)`

`A mod B` を返します。`B == 0` のとき `"0"` を返します。

---

### 5.4 比較演算

#### `(= A B)`

A と B が等しければ `"1"`、そうでなければ `"0"` を返します。

両オペランドが `isIntStr()` を満たす場合は **整数比較**、それ以外は **文字列比較**（大文字小文字を区別）です。

```
(= 3 3)           → "1"
(= 10 9)          → "0"
(= abc abc)       → "1"   （文字列比較）
(= ABC abc)       → "0"   （文字列比較、大文字小文字区別）
```

#### `(! A [B])`

`(! A)` — A が `"0"` のとき `"1"`、それ以外は `"0"`（論理 NOT）。  
`(! A B)` — A と B が異なるとき `"1"`、等しければ `"0"`（`=` の逆）。同じ整数/文字列判定ルールを適用。

```
(! 0)             → "1"
(! 1)             → "0"
(! 3 5)           → "1"
(! abc abc)       → "0"
```

#### `(< A B)`

A < B のとき `"1"`、そうでなければ `"0"`。常に整数比較。

#### `(> A B)`

A > B のとき `"1"`、そうでなければ `"0"`。常に整数比較。

#### `(between A B C)`

A ≤ B ≤ C のとき `"1"`、そうでなければ `"0"`。常に整数比較。

```
(between 1 (method) 3)     ; method が 1〜3 なら真
```

---

### 5.5 文字列操作

#### `(slash A)`

A の中の `\` をすべて `/` に置換した文字列を返します。Unix 系ツールに `\` を含むパスを渡すときに使用します。

```
(slash (arc))     ; "C:\work\file.rar" → "C:/work/file.rar"
```

## 6. 評価モデルの詳細

### 6.1 セクションの実行

各セクションの本体は `exec_script()` が呼び出す際に次のように前処理されます。

```cpp
// ArcB2e.cpp exec_script() より
wchar_t* script = new wchar_t[ki_strlen(scr)+8];
ki_strcpy( script, L"(exec " );
ki_strcat( script, scr );
ki_strcat( script, L")" );
rvm->eval( script );
```

セクション本体全体が `(exec ...)` でラップされ、`eval()` に渡されます。このため、セクション内のトップレベルの式は左から順に実行されます。

### 6.2 `m_Result` と終了コード

`cmd`/`xcmd` の実行結果（プロセス終了コード）は `m_Result` に格納されます。`exec_script()` の戻り値がこの値です。

- 0 以上 0x7FFF 以下: 成功/警告（Aile は操作を続行）
- 0x8000 以上: エラー（Aile は操作を中断）

```
(if (! 0 (cmd a (arc.rar) (listr)))
    (xcmd del (arc.rar)))         ; 失敗時にアーカイブを削除
```

### 6.3 引数の遅延評価

`getarg(a, b, r)` は:

1. `b` が `true`（`(...)` の形式）の場合: `eval(a)` を再帰的に呼び出し、結果を `r` に格納。
2. `b` が `false`（リテラル）の場合: `%x` 変数展開を行い、結果を `r` に格納。

`exec_function` の実装側は引数を必要なときにのみ `getarg` で評価します。これにより `(if cond then else)` の未選択ブランチは評価されません。

### 6.4 in-place パース

`split()` と `eval()` は引数文字列を in-place で書き換えます（`\0` を挿入してトークンを分割）。このため、同じ文字列バッファを 2 回 `eval()` することはできません。`while` ループがループの度にバッファをコピーするのはこの制約によるものです。

## 7. 前処理

`kl_rythp.cpp` の VM が受け取る前に、B2E スクリプトは `B2eScript.cpp` の 2 段階で前処理されます。

### 7.1 `B2e_PreprocessScriptInPlace`

- `;` から行末まで（文字列の外側のみ）を空白に置き換えます。
- 文字列は `"..."` で判定します。文字列はまたがらないため、1 行ずつ処理します。

### 7.2 `B2e_SplitSectionsInPlace`

- `load:`, `encode:`, `encode1:`, `decode:`, `sfx:`, `sfxd:`, `decode1:`, `list:`, `test:`, `delete:` を認識し、各セクションへのポインタを返します。
- `encode1:` が存在すると `sections.pack1 = true` が設定され、`aArchive` 能力フラグが無効になります。
- `sfxd:` が存在すると `sections.sfxDirect = true` が設定されます。

## 8. 文法まとめ（EBNF 風）

```ebnf
script      = { section } ;
section     = section-header , { statement } ;
section-header = "load:" | "encode:" | "encode1:" | "decode:" | "decode1:"
               | "sfx:"  | "sfxd:"   | "list:"    | "test:"   | "delete:" ;

statement   = expression ;

expression  = atom | list-expr ;
list-expr   = "(" , expr-seq , ")" ;
expr-seq    = expression , { ws , expression } ;

atom        = literal | quoted-string ;
literal     = (* 空白・"("・")"・";" を含まない 1 文字以上の文字列 *) ;
quoted-string = '"' , { qchar } , '"' ;
qchar       = (* '"' 以外の文字, または "%" に続く任意の 1 文字 *) ;

ws          = whitespace | comment ;
whitespace  = " " | "\t" | "\r" | "\n" ;
comment     = ";" , { (* 行末以外 *) } , "\n" ;
```

## 9. 既知の制限事項

| 制限 | 内容 |
|---|---|
| 変数は 1 文字 | 変数名として有効なのは先頭の 1 文字だけ。利用可能な変数は 52 個（a-z, A-Z）。 |
| 整数は C `int` 幅 | 32 ビット符号付き整数。ファイルサイズなど大きな値は `(size ...)` の返り値を直接使わず、ゼロとの比較のみに留めることを推奨。 |
| 文字列は行をまたがれない | `"..."` は 1 行内に収める。 |
| in-place 評価 | 同じバッファを 2 回 `eval()` できない。`while` を除き、スクリプト作者には透過的。 |
| カンマ入り数値 | `"1,234"` は `isIntStr()` が `true` を返すが `getInt()` は `1` を返す。ゼロとの比較以外の用途では正しく動作しない。 |
| ゼロ除算 | `(/ A 0)` は `A` を返す、`(mod A 0)` は `0` を返す（エラーにならない）。 |
| 文字列としての大文字小文字 | `(=)` と `(!)` の文字列比較は大文字小文字を区別する。 |
